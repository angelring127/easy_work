# 운영 패턴 구현 Todo

이 문서는 운영 패턴 1차 구현의 진행 상태를 추적하는 체크리스트다.

- 기능 기준: [운영 패턴 기능 명세서](./operating-pattern-feature-spec.md)
- 기술 설계: [운영 패턴 구현 계획](./operating-pattern-implementation-plan.md)
- 상위 기능: [자동 스케줄 기능 명세서](./auto-schedule-feature-spec.md)

## 진행 규칙

- 체크박스는 실제 구현과 검증이 끝난 뒤에만 완료 처리한다.
- 각 단계의 완료 조건을 충족하기 전에는 다음 단계의 완료 처리를 하지 않는다.
- 데이터 모델, API, UI, 자동 배정 엔진 순서로 구현한다.
- 기존 자동 배정 동작은 운영 패턴이 없는 요일의 fallback으로 유지한다.
- 모든 사용자 표시 문구는 한국어, 영어, 일본어 번역을 함께 추가한다.

## 4. 자동 배정 엔진

- [ ] 자동 배정 대상 기간의 운영 패턴 조회
- [ ] 날짜의 요일에 맞는 활성 운영 패턴 선택
- [ ] 운영 패턴이 없는 날짜는 기존 로직으로 fallback
- [ ] 시간 구간과 겹치는 배정의 고유 직원 수 계산
- [ ] 배정 직원들의 전체 보유 역할 계산
- [ ] 복수 역할 직원이 여러 필요한 역할을 동시에 충족하도록 처리
- [ ] 구간별 부족 인원과 미충족 역할 계산
- [ ] 부족 구간을 완전히 덮는 기존 근무항목 후보 조회
- [ ] 구간 대비 초과 시간이 적은 근무항목 우선 처리
- [ ] 희망 일 근무 시간과 근무항목 길이 차이를 점수에 반영
- [ ] 기존 필수 조건 필터 유지
- [ ] 기존 조건 우선순위 점수 유지
- [ ] 직원 배정 우선순위 보조 점수 유지
- [ ] 주간/월간 근무시간 공정성 tie-breaker 유지
- [ ] 한 직원에게 하루 한 개 근무항목만 배정하는 규칙 유지
- [ ] 긴 근무항목이 여러 구간을 동시에 충족하도록 계산
- [ ] 운영 패턴이 있는 날짜에는 별도 1시간 오프닝 슬롯을 생성하지 않음
- [ ] 더 이상 배정할 수 없을 때 무한 반복 없이 미충족 결과 생성

완료 조건:

- [ ] 운영 패턴이 없는 기존 매장의 자동 배정이 계속 동작한다.
