# 운영 패턴 기능 명세서

이 문서는 Workeasy 자동 스케줄의 `운영 패턴` 기능을 정의한다. 운영 패턴은 매장 영업 중 각 시간대에 최소 몇 명이 필요하고, 팀 전체가 어떤 역할을 커버해야 하는지를 설정하는 기준이다.

1차 목표는 기존 영업시간 설정을 대체하는 것이 아니라, 자동 배정이 시간대별 인원 부족을 정확히 판단하도록 만드는 것이다.

관련 문서:

- [운영 패턴 구현 계획](./operating-pattern-implementation-plan.md)
- [운영 패턴 구현 Todo](./operating-pattern-todo.md)

## 1. 배경

기존 자동 배정은 오전/오후 인원을 큰 단위로 계산한다. 이 방식은 다음 문제가 있다.

- `오후 인원 3명`처럼 보이지만 실제 특정 시간대에는 2명만 일할 수 있다.
- `Opening 10:00-22:30`, `Half time 11:00-17:30`, `Closing 17:30-24:00`처럼 긴 근무항목과 짧은 근무항목이 섞이면 실제 커버리지 공백이 잘 드러나지 않는다.
- 오프닝/오전/오후/클로징을 각각 별도 예외 로직으로 다루면 매장마다 다른 운영 방식을 표현하기 어렵다.

운영 패턴은 이 문제를 `시간 구간별 필요 조건`으로 풀기 위한 모델이다.

## 2. 핵심 개념

### 2.1 운영 패턴

운영 패턴은 반복되는 하루 운영 형태다.

예시:

- 평일 패턴
- 금요일 패턴
- 주말 패턴

각 운영 패턴은 적용 요일과 여러 시간 구간을 가진다.

### 2.2 시간 구간

시간 구간은 해당 시간대에 필요한 최소 조건이다.

1차 UI는 다음 세 항목만 보여준다.

| 항목 | 설명 |
| --- | --- |
| 시간 | 시작 시간과 종료 시간 |
| 최소 인원 | 해당 시간대에 실제로 근무해야 하는 최소 사람 수 |
| 필요한 역할 | 팀 전체가 커버해야 하는 역할 체크박스 |

사용자에게는 단순히 `시간 / 최소 인원 / 필요한 역할`로 보여준다.

내부적으로는 선택된 역할을 각각 최소 1명 필요하다고 해석한다. 단, 한 직원이 여러 역할을 보유하면 여러 역할 조건을 동시에 만족할 수 있다.

## 3. 대표 운영 예시

Granville 같은 매장의 기본 패턴은 아래처럼 표현할 수 있다.

| 구간 | 이름 | 최소 인원 | 필요한 역할 |
| --- | --- | ---: | --- |
| 10:00-11:00 | 오프닝 | 1명 | 수퍼바이저 |
| 11:00-17:30 | 오전 | 2명 | 수퍼바이저, 키친, 캐셔 |
| 17:30-22:00 | 오후 | 3명 | 수퍼바이저, 키친, 캐셔 |
| 22:00-00:00 | 클로징 | 2명 | 키친, 캐셔 |

`11:00-17:30` 구간은 총 2명이지만 필요한 역할은 3개다. 이때 수퍼바이저가 키친 또는 캐셔도 할 수 있다면 2명으로도 조건을 충족할 수 있다.

예시:

| 직원 | 보유 역할 | 결과 |
| --- | --- | --- |
| A | 수퍼바이저, 캐셔 | 수퍼바이저와 캐셔 조건 충족 |
| B | 키친 | 키친 조건 충족 |

이 팀은 총 2명이고, 수퍼바이저/키친/캐셔 역할이 모두 커버되므로 `11:00-17:30` 조건을 충족한다.

## 4. 적용 요일

1차 구현은 요일 기반 운영 패턴만 지원한다.

규칙:

- 운영 패턴은 여러 개 만들 수 있다.
- 각 운영 패턴은 적용 요일을 선택할 수 있다.
- 각 요일은 하나의 운영 패턴에만 배정될 수 있다.
- 같은 요일을 여러 패턴에 중복 선택하려 하면 저장할 수 없다.

예시:

| 패턴 | 적용 요일 |
| --- | --- |
| 평일 | 월, 화, 수, 목 |
| 금요일 | 금 |
| 주말 | 토, 일 |

특정 날짜 예외 패턴은 1차 범위에서 제외한다. 다만 이후 확장 가능하도록 데이터 모델과 API는 예외일 추가를 방해하지 않는 방향으로 설계한다.

## 5. 기존 근무항목과의 관계

운영 패턴은 `무엇을 채워야 하는지`를 정의한다.

실제 스케줄 배정은 항상 기존 `근무항목` 단위로 생성한다.

예:

- 운영 패턴: `10:00-11:00 / 최소 1명 / 수퍼바이저`
- 실제 배정 근무항목: `Opening 10:00-22:30`

이 경우 `Opening 10:00-22:30`을 수퍼바이저 역할을 가진 직원에게 배정하면 10:00-11:00 오프닝 구간을 충족한다.

자동 배정은 운영 패턴 구간을 직접 스케줄로 만들지 않는다. 기존 `Opening`, `Half time`, `Closing`, `Full time` 같은 근무항목을 조합해 운영 패턴을 만족시키려 한다.

## 6. 역할 커버리지 규칙

운영 패턴의 `필요한 역할`은 직책별 고정 담당자를 뜻하지 않는다. 팀 전체가 해당 역할을 수행할 수 있는지를 뜻한다.

규칙:

- 직원은 여러 역할을 가질 수 있다.
- 한 직원은 실제 인원으로는 1명만 센다.
- 한 직원이 여러 역할을 보유하면 여러 역할 조건을 동시에 충족할 수 있다.
- 선택된 역할은 각각 최소 1명 이상 커버되어야 한다.

예시:

조건:

- 최소 인원 2명
- 필요한 역할: 수퍼바이저, 키친, 캐셔

충족 사례:

- A: 수퍼바이저, 캐셔
- B: 키친

불충족 사례:

- A: 수퍼바이저
- B: 키친

두 번째 사례는 총 인원 2명은 충족하지만 캐셔 역할이 커버되지 않는다.

## 7. 자동 배정 우선순위

자동 배정은 운영 패턴에서 부족한 구간을 찾고, 그 구간을 덮을 수 있는 기존 근무항목을 배정한다.

근무항목 선택 원칙:

1. 부족 구간을 실제로 덮을 수 있어야 한다.
2. 부족 구간을 덮으면서 초과 시간이 가장 적은 근무항목을 우선한다.
3. 직원의 희망 일 근무 시간과 근무항목 길이가 가까울수록 점수를 더 준다.
4. 기존 필수 조건을 통과해야 한다.

예:

부족 구간이 `11:00-17:30`이면 가능한 근무항목은 다음처럼 평가한다.

| 근무항목 | 시간 | 평가 |
| --- | --- | --- |
| Half time | 11:00-17:30 | 초과 시간이 없어 우선 |
| Full time | 11:00-24:00 | 필요 구간보다 길어 후순위 |
| Opening | 10:00-22:30 | 필요 구간보다 앞뒤로 길어 후순위 |

## 8. 자동 배정 결과 경고

1차에서는 운영 패턴을 완전히 충족하지 못해도 생성된 스케줄을 저장한다. 대신 결과에 미충족 구간을 명확히 표시한다.

경고 예시:

- `화 17:30-22:00: 총 인원 3명 중 2명만 배정`
- `화 17:30-22:00: 캐셔 역할 미충족`
- `일 22:00-00:00: 키친 역할 미충족`

저장 차단은 후속 옵션으로 남긴다. 예를 들어 나중에 매장 설정에서 `미충족 시 저장 차단`을 켤 수 있게 한다.

## 9. UI 요구사항

운영 패턴 설정은 1차에서 `매장 설정 > 자동 스케줄` 안에 추가한다.

### 9.1 패턴 목록

표시 항목:

- 패턴 이름
- 적용 요일
- 시간 구간 수
- 활성 여부

기본 동작:

- 새 패턴 추가
- 패턴 이름 수정
- 적용 요일 선택
- 패턴 삭제 또는 비활성화

### 9.2 시간 구간 편집

구간 입력 항목:

- 구간 이름
- 시작 시간
- 종료 시간
- 최소 인원
- 필요한 역할 체크박스

역할 체크박스는 기존 매장 직무 역할(`store_job_roles`)을 사용한다.

### 9.3 검증 메시지

저장 전 검증:

- 같은 요일이 여러 패턴에 중복 선택되면 저장 불가
- 한 패턴 안에서 시간 구간이 겹치면 저장 불가
- 구간 시작 시간이 종료 시간보다 늦거나 같으면 저장 불가
- 최소 인원은 1명 이상이어야 함
- 필요한 역할이 없어도 저장은 가능하지만 경고 표시

1차에서는 시간 구간 사이의 빈틈은 경고로 표시한다. 매장에 따라 실제로 빈 구간을 의도할 수 있기 때문이다. 다만 자동 배정 검증에서는 빈 구간이 있으면 해당 시간대는 커버리지 기준이 없는 것으로 본다.

## 10. 범위

### 10.1 1차 포함

- 여러 운영 패턴 생성
- 패턴별 적용 요일 선택
- 요일 중복 방지
- 시간 구간 관리
- 구간별 최소 인원 설정
- 구간별 필요한 역할 체크박스
- 자동 배정에서 운영 패턴 기준으로 미충족 구간 계산
- 미충족 구간 경고 표시
- 기존 근무항목만 사용해 배정

### 10.2 1차 제외

- 특정 날짜 예외 패턴
- 운영 패턴이 기존 영업시간 설정을 대체하는 기능
- 자동 근무항목 생성
- 미충족 시 저장 차단
- 역할별 최소 인원 수량 직접 입력
- 시간 구간별 최대 인원

## 11. 완료 기준

운영 패턴 1차는 다음을 만족하면 완료로 본다.

- 매장 설정 > 자동 스케줄에서 운영 패턴을 만들 수 있다.
- 각 패턴에 적용 요일을 선택할 수 있다.
- 각 요일은 하나의 패턴에만 배정된다.
- 패턴 안에 시간 구간을 추가/수정/삭제할 수 있다.
- 구간마다 최소 인원과 필요한 역할을 설정할 수 있다.
- 자동 배정은 적용 요일에 맞는 운영 패턴을 찾아 사용한다.
- 자동 배정은 기존 근무항목만 생성한다.
- 자동 배정 결과에 미충족 시간 구간과 미충족 역할이 표시된다.
- 기존 필수 조건과 점수 조건은 계속 적용된다.
- `npx tsc --noEmit`, `npm run lint`, `npm run build`가 통과한다.

## 12. 후속 확장

- 특정 날짜 예외 패턴
- 휴일/공휴일 패턴
- 운영 패턴을 영업시간 설정과 연결하거나 대체
- 미충족 시 저장 차단 옵션
- 역할별 최소 인원 수량 입력
- 시간 구간별 최대 인원
- 자동 근무항목 추천 또는 생성
